# Security: секреты, ключи, доступы

## Никогда не коммитьте секреты

Запрещено добавлять в git:
- `.env` файлы
- приватные ключи (EVM, SSH, API keys)
- seed phrases / mnemonics
- токены доступа (GitHub, облака, LLM провайдеры)

Даже в публичной экосистеме это остаётся критичным.

---

## Рекомендуемая практика

- Храните секреты в менеджере секретов или локально в `.env` (который в `.gitignore`).
- Перед push запускайте проверку:

```bash
git status
git diff
```

- Если секрет случайно попал в git:
  - немедленно **отзовите ключ**
  - перепишите историю (git filter-repo / BFG) и сделайте force-push

---

## Ответственное раскрытие

Если вы нашли уязвимость или утечку секретов:
- не публикуйте детали в публичных issue
- пришлите описание приватно (владельцам репозитория/maintainers)
